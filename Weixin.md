> 排版过的文档永不覆盖原文件，改错了随时回退。

写报告、写方案、写总结——内容写完了，格式调半天。

调完一遍，发现某段标题被识别错了，又得重新来。熟练工也要十几分钟，新手半小时打底。

## 在 AI 客户端里，一句话就够了

**docx-helper** 是一个 AI Skill，可以加载到各类 AI 客户端中使用。它按我自己常用的材料排版习惯来设置格式——参考 GB/T 9704-2012，但最终出来的效果就是一个干净、规范、看着舒服的文档。

排版后你的文档会变成这样：

- **页面**：A4，上 3.7 / 下 3.5 / 左右 2.7 cm
- **大标题**：方正小标宋 二号（22pt），居中
- **一级标题（一、二、三、）**：方正黑体 三号（16pt）
- **二级标题（（一）（二））**：方正楷体 三号
- **三级标题（1. 2.）**：方正仿宋 三号
- **正文**：方正仿宋 三号，行距固定 28.9 磅，首行缩进 2 字符
- **页码**："— N —"，四号 Times New Roman
- **西文/数字**：Times New Roman
- **封面**：自动检测并排版标题、副标题、单位、日期

更关键的是——**它不覆盖你的原文件**。每次排版输出一个带版本号的新文件：

```
报告+docx-helper+v1.docx
报告+docx-helper+v2.docx
报告+docx-helper+v3.docx
```

改坏了？直接回退到上一个版本。改完了想再调一下？基于 v1 继续迭代。全程可追溯。

## 实操演示：在 WorkBuddy 里怎么用

安装完成后，有两种方式可以调用：

**方式一：斜杠命令（最直接）**

直接在输入框输入：

> `/docx-helper 检查这份文档的格式`

Skill 立刻启动，开始分析诊断。这种方式的优点是精准——不会有歧义，100% 被触发。

**方式二：自然语言聊天**

> ? "帮我把这份报告排版"

AI 会自动读取文档，逐段分析标题层级、字体、编号是否合规，然后把诊断结果一条条列给你看：

> "第 0 段识别为大标题，将设为方正小标宋二号居中；第 3 段识别为一级标题'一、项目背景'，将设为黑体三号；第 5 段使用了 '●' 符号，参考 GB/T 9704-2012 建议改为'（一）'……"

**第二步：看一眼，确认或调整**

> ? "第 5 段的 ● 不用改，其他没问题"

AI 记住你的调整，然后执行排版。

**第三步：收文件**

几秒钟后，对话框里弹出 `报告+docx-helper+v1.docx`，下载直接用。

全程你没打开过 Word、没翻过标准文件、没调过一行格式。动手时间不超过 1 分钟。

## 也支持命令行

如果你习惯用终端，同样可以操作：

```bash
# 分析诊断
python scripts/format.py --analyze 报告.docx

# 应用排版
python scripts/format.py --apply 报告.docx
```

两种方式底层是同一套引擎，选你喜欢的方式。

## 怎么装？

如果你已经在用 WorkBuddy，安装只需一句话：

> 请阅读并安装这个skill：https://github.com/hunyed15/docx-helper

AI 会自动读取仓库内容、安装依赖、配置好一切。

唯一的额外步骤：提前把电脑装上四款方正字体（方正小标宋、黑体、楷体、仿宋_GBK），排版后的文档要用到这些字体。
